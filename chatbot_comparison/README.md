# Chatbot Comparison

Manual head-to-head vs free-tier general-purpose chatbots — the contrast that grounds the "niche value" argument in Part 3.

## Files

- `prompt.md` — paste-ready prompt + context (last 2 weeks of demand + weather, future weather for next 168h). Generated by `build_prompt.py`.
- `build_prompt.py` — regenerate `prompt.md` from the processed parquet.
- `score_chatbot.py <name> <csv_path>` — score a chatbot's CSV reply against actuals; writes `outputs/forecasts/<name>.{parquet,json}` so it lands in the metrics table on next notebook run.
- `chatgpt_freetier.md`, `claude_freetier.md` — per-chatbot transcripts + qualitative notes (create manually).
- `screenshots/` — PNGs of the chat UI.

## Protocol

1. Open ChatGPT free tier (or Claude free tier) in a fresh chat. Note model name + date.
2. Paste `prompt.md` verbatim.
3. Save full reply into `<bot>_freetier.md`; screenshot the UI into `screenshots/`.
4. Extract the 168-row forecast into a CSV with columns `timestamp,q10,q50,q90` (if no PI given, copy q50 into q10/q90 and document this).
5. Run `python chatbot_comparison/score_chatbot.py <bot> <csv>` to score.
6. Note iterations, refusals, hallucinated dates/values, runtime — these feed Part 3.

Expect free-tier chatbots to refuse, truncate, or fabricate. That's the point.
